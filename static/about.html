TL;DR: End-to-End GitOps Demo

This project is an automated, production-style pipeline demonstrating modern DevOps practices for a simple Go web application.

The core function is to implement Continuous Delivery (CD) via GitOps, ensuring every code change is automatically built, containerized, and deployed to a cloud cluster.

The Pipeline in Brief

Step Technology Role
Code Go The foundational web application.
Container Docker Packages the app into a small, secure image (multi-stage build).
CI GitHub Actions Runs tests, builds the Docker image, and updates Helm configuration in Git.
Orchestration Kubernetes (EKS) The target deployment environment.
CD (GitOps) Argo CD Monitors the Git repo and automatically syncs the new version to Kubernetes.

The entire workflow is triggered by a simple code commit, resulting in a zero-touch deployment to the cluster.
